> 그래서 게임을 가장 작게 줄이고, LLM이 추측·되묻기를 하지 않도록 명세를 다 적어 둡니다.

5. 파일은 **`main.py` 하나**.

## 1. 게임 규칙 (한 줄)

격자 위 화살표를 클릭하면, **머리 방향의 직선이 보드 끝까지 비어 있을 때** 화면 밖으로 나가 사라진다.
막혀 있으면 **빨갛게 변하고 생명 −1**. 모든 화살표를 없애면 **클리어**.

---

## 2. 구현 범위 (초소형)

**만들 것:**
- `main.py` 1개 (pygame).
- 고정 격자(줌·이동 없음). 레벨 **2개**를 코드에 내장.
- 화살표 = 칸 경로 + 머리 방향. 클릭하면 **즉시 제거**.
- 막히면 **빨강 + 생명 −1**. 생명 3개, 0이면 레벨 재시작.
- 전부 제거 → "Clear" 글자 → 클릭하면 다음 레벨(없으면 레벨1).
- 배경은 칸마다 옅은 점(점 격자).

**만들지 말 것:** 애니메이션 / 줌·드래그 / 길게 누르기 / 레벨선택 / JSON / 저장 / 사운드 / 확장 전부.

## 3. 핵심 로직 `can_exit` (이대로)

```python
def can_exit(board, arrow):
    dr, dc = arrow["dir"]
    r, c = arrow["cells"][-1]          # 머리 칸
    r, c = r + dr, c + dc
    while 0 <= r < board["rows"] and 0 <= c < board["cols"]:
        other = board["occupancy"].get((r, c))
        if other is not None and other is not arrow and not other["removed"]:
            return False               # 다른 화살표가 막음
        r, c = r + dr, c + dc
    return True                        # 보드 밖 -> 탈출 가능
```

- 클릭한 화살표가 `can_exit` 면: `removed=True` + `occupancy`에서 그 칸들 삭제.
- 아니면: `blocked=True`(빨강) + 생명 −1.

## 4. 데이터 모델 + 레벨 (그대로 복사)

방향: UP `(-1,0)`, DOWN `(1,0)`, LEFT `(0,-1)`, RIGHT `(0,1)`.
화살표 `cells`는 **꼬리 → 머리** 순서, `cells[-1]`이 머리. `dir`은 머리 방향(= 마지막 두 칸이 향하는 방향).

```python
LEVELS = [
    {   # 레벨 1 (5x5)
        "rows": 5, "cols": 5, "lives": 3,
        "arrows": [
            {"cells": [(1,0),(1,1)],            "dir": (0,1)},
            {"cells": [(3,4),(2,4),(1,4)],      "dir": (-1,0)},
            {"cells": [(2,2),(3,2)],            "dir": (1,0)},
            {"cells": [(0,3),(0,2),(0,1)],      "dir": (0,-1)},
        ],
    },
    {   # 레벨 2 (5x5, 꺾인 화살표 포함)
        "rows": 5, "cols": 5, "lives": 3,
        "arrows": [
            {"cells": [(4,4),(3,4),(2,4),(2,3)], "dir": (0,-1)},
            {"cells": [(3,0),(4,0)],             "dir": (1,0)},
            {"cells": [(2,0),(2,1),(2,2),(3,2)], "dir": (1,0)},
            {"cells": [(0,3),(1,3),(1,4)],       "dir": (0,1)},
            {"cells": [(0,1),(0,0)],             "dir": (0,-1)},
        ],
    },
]
# 로드할 때 각 화살표에 "removed": False, "blocked": False 추가
```
> 이 두 레벨은 **반드시 풀 수 있는** 배치입니다(칸 겹침 없음).

## 5. 화면 / 색상 / 입력

- 창 **480×640**. 상단에 생명(파란 원 3개).
- 셀 크기 = `min(가용너비//cols, 가용높이//rows, 80)` 로 1번 계산. 보드는 화면 중앙.
- 화살표 = 칸 중심들을 잇는 굵은 선(`pygame.draw.lines`) + 머리 삼각형. 일반 갈색, `blocked` 빨강.
- 배경 = 칸 중심마다 옅은 점(반지름 3).
- **클릭 1번 = 제거 시도.** 클릭 칸 = `((my-oy)//cell, (mx-ox)//cell)` → `occupancy`로 화살표 찾기.

```python
BG=(242,235,217); DOT=(212,200,176); ARROW=(122,92,62)
RED=(200,70,60); LIFE=(80,170,230); TEXT=(110,90,70)
```

## 완성 기준
- [ ] 창 + 점 격자 + 화살표가 보인다
- [ ] 빠질 수 있는 화살표 클릭 → 사라짐 / 막힌 것 클릭 → 빨강 + 생명 −1
- [ ] 생명 0 → 재시작, 전부 제거 → 다음 레벨
- [ ] 레벨 2개가 끝까지 풀린다
