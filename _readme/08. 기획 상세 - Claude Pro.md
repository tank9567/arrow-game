## 1. 게임 한 줄 설명

격자 위에 **여러 칸으로 이어진(꺾일 수 있는) 화살표**들이 놓여 있다.
화살표를 클릭하면 **머리(화살촉) 방향의 직선 경로가 보드 끝까지 비어 있을 때** 화면 밖으로 빠져나가 사라진다.
막혀 있으면 **빨갛게 변하고 생명이 1 줄어든다**. 모든 화살표를 없애면 **클리어**.

---

## 2. 구현 범위

**만들 것 (기본):**
- `main.py` 1개 (pygame). 고정 격자(줌/팬 없음, 화면에 맞게 셀 크기 1회 계산).
- 화살표 = 칸 경로 + 머리 방향. 클릭하면 **즉시 제거**.
- 막히면 **빨강 + 생명 −1**. 생명 3개, 0이면 레벨 재시작.
- 전부 제거 → "Clear!" → 클릭 시 다음 레벨(마지막이면 레벨1).
- 레벨 **3개**를 코드에 내장. 배경은 점 격자.

**기본에서는 빼고, [10장] 확장으로 미루기:** 슬라이드 애니메이션 / 줌·드래그 / 롱프레스 미리보기 / 레벨선택 화면 / JSON·저장 / 사운드.

---

## 3. 좌표계와 방향

- 칸 `(row, col)`. 방향 `(dr, dc)`: UP `(-1,0)`, DOWN `(1,0)`, LEFT `(0,-1)`, RIGHT `(0,1)`.
- 칸 중심 픽셀 = `(ox + col*cell + cell//2, oy + row*cell + cell//2)`
- 클릭 픽셀 → 칸 = `((my-oy)//cell, (mx-ox)//cell)`

## 4. 데이터 모델 (이 구조 그대로)

```python
arrow = {
    "cells": [(2,0),(2,1),(2,2)],  # 꼬리 -> 머리, cells[-1]=머리
    "dir":   (0,1),                # 머리 방향 (= 마지막 두 칸이 향하는 방향)
    "removed": False,
    "blocked": False,
}
# board = {"rows":R, "cols":C, "arrows":[...], "occupancy": {(r,c): arrow}}
```

## 5. 핵심 로직 `can_exit` (가장 중요)

```python
def can_exit(board, arrow):
    """머리에서 dir 방향 직선이 보드 밖까지 비면 True."""
    dr, dc = arrow["dir"]
    r, c = arrow["cells"][-1]
    r, c = r + dr, c + dc
    while 0 <= r < board["rows"] and 0 <= c < board["cols"]:
        other = board["occupancy"].get((r, c))
        if other is not None and other is not arrow and not other["removed"]:
            return False
        r, c = r + dr, c + dc
    return True
```
- 성공: `removed=True` + `occupancy`에서 그 칸들 삭제.
- 막힘: `blocked=True` + 생명 −1.

**클릭한 화살표 찾기 (넉넉한 터치):** 점유 칸이면 그 화살표, 아니면 **반경 1칸 안에서 가장 가까운** 화살표.
```python
def arrow_at(board, mx, my, ox, oy, cell):
    col, row = (mx-ox)//cell, (my-oy)//cell
    a = board["occupancy"].get((row, col))
    if a and not a["removed"]:
        return a
    best, bd = None, cell*1.0
    for a in board["arrows"]:
        if a["removed"]: continue
        for (r,c) in a["cells"]:
            cx, cy = ox+c*cell+cell//2, oy+r*cell+cell//2
            d = ((mx-cx)**2+(my-cy)**2)**0.5
            if d <= bd: bd, best = d, a
    return best
```

## 6. 화면 구성

- 창 **480×640**. 상단 60px에 생명(파란 원 3개) + 레벨 번호.
- 셀 크기 = `min(가용너비//cols, 가용높이//rows, 80)` 1회 계산. 보드 중앙 배치.
- 화살표: 칸 중심들을 잇는 굵은 선 + 머리 삼각형. 일반 갈색, `blocked` 빨강.
- 배경: 칸 중심마다 옅은 점(반지름 3~4).
- 클리어: 중앙 "Clear!" → 클릭 시 다음 레벨. 생명 0: "Game Over" → 클릭 시 재시작.

```python
BG=(242,235,217); DOT=(212,200,176); ARROW=(122,92,62)

```

## 7. 입력 / 흐름

- 마우스 클릭 1번 = 화살표 제거 시도.
- 상태: `playing → clear`(전부 제거) / `playing → gameover`(생명 0).
- `clear` 클릭 → 다음 레벨(마지막이면 레벨1). `gameover` 클릭 → 같은 레벨 재시작.

## 10. 확장 (여유가 있으면 1개씩)

받은 코드가 잘 돌아가면, **한 번에 하나씩** 추가 요청하세요:
1. **막힘 피드백 강화**: 막힌 화살표를 빨갛게 + 잠깐 흔들리게(0.2초).
2. **빠짐 애니메이션(쉬운 버전)**: 제거되는 화살표를 머리 방향으로 몇 프레임 밀어내며 사라지게.
3. **레벨 추가**: 8장 형식으로 1~2개 더(칸 겹치지 않게).
4. **길게 누르기 미리보기**: 누르고 있으면 머리 방향 직선을 옅은 선으로 표시.
5. **줌/드래그**: 큰 보드용. (가장 토큰을 많이 쓰므로 맨 마지막.)

> 한 번에 2개 이상 요청하면 토큰이 빠르게 소모됩니다. **반드시 하나씩.**
