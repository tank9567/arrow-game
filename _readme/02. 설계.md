## 3. 좌표계 & 그리드

- 보드는 **정사각형 격자(Grid)**. 각 칸은 `(row, col)` 정수 좌표.
- 픽셀 ↔ 격자 변환:

```python
px = origin_x + col * CELL_SIZE     # 격자 -> 픽셀(좌상단)
py = origin_y + row * CELL_SIZE
col = (mouse_x - origin_x) // CELL_SIZE   # 픽셀 -> 격자
row = (mouse_y - origin_y) // CELL_SIZE
```

- 방향 (`dr`, `dc`):

| 방향 | (dr, dc) |
|------|----------|
| UP    | (-1,  0) |
| DOWN  | (+1,  0) |
| LEFT  | ( 0, -1) |
| RIGHT | ( 0, +1) |

## 4. 핵심 데이터 모델 (개정)

### 4-1. Arrow = 경로(cells) + 머리 방향
- 화살표 하나가 **여러 칸(셀)을 잇는 경로**를 가진다.
- `cells` 는 **꼬리 → 머리 순서**의 셀 목록. `cells[-1]` 이 **머리(화살촉) 칸**.
- `direction` 은 머리가 **나아갈 방향**(보통 마지막 세그먼트의 연장 방향).

```python
from enum import Enum

class Direction(Enum):
    UP = (-1, 0); DOWN = (1, 0); LEFT = (0, -1); RIGHT = (0, 1)
    @property
    def delta(self): return self.value

class Arrow:
    def __init__(self, cells, direction: Direction):
        self.cells = [tuple(c) for c in cells]   # [(r,c), ...] 꼬리->머리
        self.direction = direction
        self.is_removed = False
        self.is_blocked = False

    @property
    def head(self):
        return self.cells[-1]                     # 머리 칸
```

### 4-2. Board = 점유맵
- 어떤 칸을 어떤 화살표가 차지하는지 **점유맵(occupancy)** 으로 관리한다.
- 화살표 1개가 여러 칸을 차지하므로, 모든 셀을 펼쳐서 매핑한다.

```python
class Board:
    def __init__(self, rows, cols, arrows):
        self.rows, self.cols, self.arrows = rows, cols, arrows
        # {(r,c): Arrow}  화살표가 차지한 모든 칸
        self.occupancy = {}
        for a in arrows:
            for cell in a.cells:
                self.occupancy[cell] = a
```

> **학습 포인트**: "객체 1개 ↔ 칸 1개"가 아니라 "객체 1개 ↔ 칸 여러 개"로 바뀐 점. 점유맵으로 충돌·제거를 O(1) 조회한다.

## 5. 핵심 알고리즘: 충돌 판정 (기차처럼 탈출)

머리에서 **진행 방향 직선**을 따라가며, 보드 밖에 닿기 전에 **다른 화살표**가 막으면 탈출 불가.

```python
def can_exit(self, arrow) -> bool:
    """머리의 진행 방향 직선 경로가 보드 밖까지 비어 있으면 True."""
    dr, dc = arrow.direction.delta
    r, c = arrow.head
    r, c = r + dr, c + dc                 # 머리 다음 칸부터
    while 0 <= r < self.rows and 0 <= c < self.cols:
        other = self.occupancy.get((r, c))
        if other is not None and other is not arrow and not other.is_removed:
            return False                  # 다른 화살표가 가로막음
        r, c = r + dr, c + dc
    return True                           # 보드 밖에 도달 -> 탈출 가능
```

- `other is not arrow`: 머리 직선상에 **자기 몸통**이 있어도 무시(자기 자신은 막지 않음).

### 제거 / 판정
```python
def try_remove(self, arrow) -> bool:
    if self.can_exit(arrow):
        for cell in arrow.cells:          # 점유 해제(칸 비우기)
            self.occupancy.pop(cell, None)
        arrow.is_removed = True
        return True
    arrow.is_blocked = True               # 막힘 -> 빨강 (생명 차감은 호출부)
    return False

def is_cleared(self):
    return all(a.is_removed for a in self.arrows)
```

### 클릭한 칸 -> 화살표
```python
def arrow_at_pixel(self, x, y):
    cell = self.pixel_to_cell(x, y)       # 머리든 몸통이든 클릭된 칸이 속한 화살표
    a = self.occupancy.get(cell) if cell else None
    return a if (a and not a.is_removed) else None
```

> **학습 포인트**: 충돌 판정 자체는 "방향 벡터를 반복해 더하며 검사"하는 단순 루프. 핵심은 화살표가 여러 칸을 점유한다는 점과 *자기 자신 제외* 처리.

## 6. 렌더링 (점 격자 + 경로형 화살표)

> ⚠️ **개정**: 배경을 바둑판 격자선이 아니라 **회색 점 격자(dot grid)** 로 그린다. (샘플 앱과 동일)

- **배경 = 점 격자**: 각 칸 중심(`cell_center`)에 옅은 회색 점을 찍는다. 화살표는 이 점들을 잇는 선처럼 보인다.
- **화살표 = 점을 잇는 경로**: 셀 중심들을 잇는 **굵은 폴리라인(꼬리)** + **머리 칸의 화살촉(삼각형)**.
- 화살표가 빠져나가면 그 자리엔 **점만 남아** 자연스러운 "빈 자리 흔적"이 된다. (별도 점선 그리기 불필요)

```python
def draw_dots(screen, board):
    for r in range(board.rows):
        for c in range(board.cols):
            pygame.draw.circle(screen, COLOR_DOT, board.cell_center(r, c), DOT_RADIUS)

def draw_arrow(screen, arrow, board):
    color = RED if arrow.is_blocked else BROWN
    pts = [board.cell_center(r, c) for (r, c) in arrow.cells]
    if len(pts) >= 2:
        pygame.draw.lines(screen, color, False, pts, width=8)   # 꼬리(꺾인 선)
    draw_head(screen, arrow.head, arrow.direction, color)        # 머리 화살촉
```

- `draw_head`: 머리 칸 중심에서 방향으로 삼각형(앞 뾰족 + 좌우 날개).

### 6-1. 탈출/막힘 애니메이션
- **탈출 성공**: 화살표가 자기 경로(폴리라인) + 머리 방향 연장선을 따라 **화면 밖으로 미끄러져** 나간다(기차). 폴리라인 arc-length로 `[꼬리, 머리]` 구간을 이동시키며 그린다.
- **막힘(실패)**: 머리가 진행 방향으로 **장애물 직전까지 전진 → 부딪히는 순간 빨간색으로 변함 → 제자리로 복귀**. 복귀 후 생명 1 차감.
- 애니메이션 동안에는 입력을 잠근다.

## 7. 게임 상태 관리 (상태 머신)

```python
class GameState(Enum):
    HOME = "home"; PLAYING = "playing"; CLEAR = "clear"; GAMEOVER = "gameover"
```

```
HOME ─[플레이]→ PLAYING ─[모든 화살표 제거]→ CLEAR ─[다음 레벨]→ PLAYING
                  └─[생명 0]→ GAMEOVER ─[다시하기]→ PLAYING
   (CLEAR/GAMEOVER) ─[홈]→ HOME
```

## 8. 입력 처리 (탭 vs 롱프레스)

| 입력 | 동작 |
|------|------|
| **짧은 클릭(탭)** | 클릭된 칸의 화살표 빼내기 시도 → 성공/실패 |
| **길게 누르기(롱프레스)** | 그 화살표 머리의 진행 방향을 화면 끝까지 선으로 미리보기 |

```python
LONG_PRESS_MS = 300
# DOWN: press_arrow=arrow_at_pixel(pos), press_start=now
# 누르는 중 (now-press_start>=LONG_PRESS_MS) 면 머리 직선 경로 미리보기
# UP: 경과<LONG_PRESS_MS 면 탭 -> try_remove
```

- 미리보기 선 색상: 일반=기본/초록, `is_blocked`=빨강.

## 9. 레벨 데이터 포맷 (개정: 경로 기반)

```json
{
  "level": 1,
  "name": "튜토리얼",
  "rows": 5,
  "cols": 5,
  "lives": 3,
  "arrows": [
    { "cells": [[2,0],[2,1],[2,2]], "dir": "RIGHT" },
    { "cells": [[0,4],[1,4],[2,4]], "dir": "DOWN"  },
    { "cells": [[4,1],[4,2]],       "dir": "LEFT"  }
  ]
}
```

- `cells`: **꼬리 → 머리** 순서의 칸 목록. (인접 칸은 상하좌우로만 이어진다)
- `dir`: 머리가 나아갈 방향. 보통 `cells[-1] - cells[-2]` 방향과 같다.
- **레벨 제작 규칙**: 화살표끼리 **칸이 겹치지 않게** 배치하고, 풀이 가능(어떤 순서로든 모두 빠짐)하도록 설계 → *솔버로 검증*.

> **풀이 가능성 검증(솔버)**: "지금 빠질 수 있는(can_exit) 화살표를 계속 제거" 반복 후 전부 제거되면 풀이 가능. 제거는 다른 화살표의 진로를 막지 않으므로 greedy로 판정 가능.
