# 화살표 게임 (Arrows GO!) 개발 진행 문서

> [02. 설계.md](02.%20설계.md) 를 기반으로 한 **MVP 개발 진행 체크리스트**.
> 각 단계를 완료할 때마다 체크하며, 단계가 끝나면 실행/확인 후 다음 단계로 넘어간다.

> 🔄 **모델 개정 반영**: 화살표가 "한 칸 방향" → **여러 칸 꺾인 경로(cells) + 화살촉(머리)** 으로 변경됨.
> 이에 맞춰 `arrow.py`(경로+화살촉 렌더), `board.py`(점유맵+머리 직선 충돌), `level.py`/레벨 JSON(경로 기반), `play_scene`(미리보기 머리 기준)을 재구현하고 전 단계 재검증 완료.
> - 화살표 = `cells`(꼬리→머리 셀 목록) + `dir`(머리 방향), 머리 = `cells[-1]`
> - 충돌 판정 = 머리의 진행 방향 직선이 보드 밖까지 비어 있으면 탈출(기차처럼)
> - 레벨 3개 재작성 + 셀 겹침 없음 + greedy 솔버로 풀이 가능성 검증
> - 꺾인 화살표(ㄱ/ㄷ/계단) 폴리라인 렌더 시각 확인

## 최종 파일 구조
```
ArrowGame/
├── main.py                 # 진입점 (게임 루프)
├── config.py               # 상수/색상/폰트 경로
├── save.json               # 진행 저장 (실행 시 생성)
├── game/
│   ├── arrow.py            # Arrow, Direction
│   ├── board.py            # Board: 좌표변환 + 충돌판정(can_exit) + 제거
│   ├── level.py            # JSON 레벨 로딩 + build_board
│   ├── progress.py         # 진행 저장/불러오기
│   ├── state.py            # GameState Enum
│   └── app.py              # Game: 진행/씬 전환 컨트롤러
├── scenes/
│   ├── scene.py            # Scene 기반 클래스
│   ├── ui.py               # 폰트/버튼/물방울 UI 헬퍼
│   ├── home_scene.py       # 홈 화면
│   ├── play_scene.py       # 플레이 화면 (탭/롱프레스)
│   └── result_scene.py     # 클리어/게임오버 화면
└── levels/
    ├── level_01.json       # 튜토리얼 (5x5) - 경로형 화살표
    ├── level_02.json       # ㄱ자 길 (6x6)
    └── level_03.json       # 얽힌 길 (7x7)
```
> 화살표 데이터는 `cells`(셀 경로) + `dir`(머리 방향) 형식. 예: `{ "cells": [[2,0],[2,1],[2,2]], "dir": "RIGHT" }`

## 진행 로그
- 1단계: `config.py`, `main.py`(창+격자) — 창/격자 렌더 확인
- 2단계: `game/arrow.py`(Direction/Arrow) — 4방향 + 빨강 렌더 스크린샷 확인
- 3단계: `game/board.py`(좌표변환/클릭탐지) — 좌표 왕복/탐지 단위테스트
- 4단계: `Board.can_exit/try_remove/is_cleared` — 충돌판정/연쇄풀이 단위테스트 (핵심)
- 5단계: 생명 차감 + 물방울 UI — 렌더 스크린샷 확인
- 6단계: `state.py`, `scenes/*`, `app.py` — 홈→플레이→클리어/게임오버 전환 + 한글폰트 통합테스트
- 7단계: JSON 레벨 3개 + `level.py`/`progress.py` — 전 레벨 풀이가능성(greedy 솔버) + 진행 저장 검증
- 8단계: 탭/롱프레스 구분 + 경로 미리보기 — 입력구분/미리보기좌표/스크린샷 확인
- 통합: 전체 컴파일 + `main()` 실행 경로 스모크 통과

### 추가 작업 (플레이 피드백 반영)
- **레벨 선택 화면**(`scenes/level_select_scene.py`): 홈 → 플레이 → 레벨(1·2·3) 선택 → 시작. (기존엔 저장된 레벨만 시작됐음)
- **기차 슬라이드 애니메이션**(`play_scene.py`): 빼낸 화살표가 즉시 사라지지 않고 **경로(폴리라인)를 따라 화면 밖으로 미끄러져** 나간다. 폴리라인 arc-length 유틸(`_point_along`/`_subpath`)로 구현, 애니메이션 중 입력 잠금.
- **데이터 검증**(`level.py _validate_arrow`): cells가 상하좌우로 인접하는지 + **화살촉 방향(dir)이 경로 끝 방향과 일치하는지** 로딩 시 확인 → 화살표 뒤집힘 데이터 오류를 막음.
- **버그 수정**: 막힌 화살표 탭 시 빨강(`is_blocked`) 표시가 누락됐던 것 수정. 레벨1 화살표 cells 순서(뒤집힘) 교정.

### 추가 작업 2 (점 격자 + 꺾인 화살표 + 막힘 애니메이션)
- **점 격자(dot grid) 렌더링**(`board.draw_dots`): 바둑판 격자선 → **각 칸 중심에 옅은 회색 점**. 샘플 앱과 동일하게, 화살표가 점들을 잇는 형태로 보이고 빠진 자리엔 점만 남아 흔적이 됨. (`config.COLOR_DOT`, `DOT_RADIUS`)
- **꺾인 화살표 레벨**: 레벨 1·2·3을 ㄱ자/ㄷ자/계단형 경로가 포함되도록 재설계. (셀 겹침 없음 + dir 일치 + greedy 솔버 풀이가능 검증)
- **막힘 부딪힘 애니메이션**(`play_scene._start_blocked_anim`): 막힌 화살표 탭 시 머리가 **장애물 직전까지 전진 → 부딪히는 순간 빨강 → 제자리로 복귀** 후 생명 -1. (성공 exit 애니메이션과 통합, 폴리라인 arc-length 활용)

### 추가 작업 3 (10단계 레벨 + 줌/팬 카메라)
- **레벨 생성기**(`tools/generate_levels.py`): 역방향 구성(머리 경로가 안 막히게 화살표를 하나씩 추가 → 추가 역순으로 제거하면 항상 풀림)으로 **풀이 보장** 레벨을 자동 생성. 단계가 오를수록 격자(5×5→9×9)·꼬리 길이·꺾임·화살표 수 증가. → `levels/level_01~10.json` 10단계.
- **줌/팬 카메라**(`play_scene`): 셀 크기는 고정하고, 보드를 자기 surface(월드 좌표, 0 기준)에 그린 뒤 **scale(줌)·pan(이동)** 을 적용해 표시. 큰 보드는 화면보다 커도 **마우스 휠로 확대/축소, 드래그로 이동**해서 본다(샘플 앱 방식). 작은 보드는 fit 배율로 화면에 맞춰 보여줌.
- **입력 구분**: 짧은 클릭(이동 없음)=탭(빼내기) / 길게 누름=경로 미리보기 / 드래그=팬 / 휠=줌. 좌표는 화면→월드 역변환 후 셀 판정.
- `board` 는 셀 고정·(0,0) 기준·`draw(surface)` 로, `arrow.draw` 는 셀 크기를 받아 선/화살촉을 비례 렌더하도록 변경.
